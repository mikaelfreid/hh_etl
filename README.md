# HH.ru Incremental ETL Project

## Описание

Этот проект реализует инкрементальный ETL для сбора вакансий с сайта hh.ru, сохранения их в PostgreSQL и экспорта в файловую систему в колоночном формате (Parquet, сжатие Snappy).

### Основные цели проекта:

Автоматически забирать только новые вакансии с сайта hh.ru.

Сохранять данные в БД для дальнейшего анализа.

Сохранять данные в Parquet для удобного использования в аналитике.

Логирование работы ETL и обработка ошибок с ретраями.

Вынесение конфигурации отдельно от кода для удобного изменения параметров.

Генерация уникального идентификатора ETL (etl_id) для отслеживания выполнения и привязки данных к конкретному запуску.

## Реализованные требования

### 1. Базовый ETL (инкремент с источника, сохранение в БД)

Используется таблица PostgreSQL etl.hh_search.

При каждом запуске определяется High Water Mark (HWM) — дата последней публикации вакансии в базе:

`df = pl.read_database("SELECT MAX(publication_time) AS hwm FROM etl.hh_search", con)`

В базу записываются только вакансии с публикацией позже HWM.
Таблица создаётся один раз при первом запуске `(init_db())`, новые данные добавляются через append.

### 2. Логирование

Настроено через стандартный модуль logging.
Логи включают:

Уникальный etl_id для запуска.

Статус каждой страницы (page_idx) и количество новых записей.

Ошибки и исключения.

### 3. Обработка ошибок (ретраи)

Для всех HTTP-запросов используется tenacity с повторными попытками и фиксированным ожиданием между ними.

Логируются все попытки и конечные ошибки:

`@retry_http
def fetch_page(url: str) -> BeautifulSoup:
    ...`
    
### 4. Вынос конфигурации отдельно от кода

Все параметры проекта (поиск, HTTP, база данных, логирование, retry) находятся в config.yaml.

### 5. Генерация уникального etl_id и прокидывание его в логи и данные

На каждый запуск создаётся уникальный идентификатор:

`etl_id = str(uuid.uuid4())`

Используется в логах, а также записывается в базу и Parquet.

### 6. Экспорт данных в файловую систему (Parquet, Snappy)

Все новые вакансии сохраняются в папку data/ в формате Parquet с сжатием Snappy:

`df_all.write_parquet(output_path, compression="snappy")`

В репозитории добавлен пример:

hh_search_demo.parquet — демонстрационный файл с результатами ETL.
